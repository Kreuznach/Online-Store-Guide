# 작업 요청: 통신판매업/스마트스토어 창업 준비 가이드 원페이지 서비스 제작

너는 VS Code Agent로 동작하는 프론트엔드 개발자이자 정보성 서비스 기획 보조자다.  
공식 출처 기반의 “통신판매업/스마트스토어 창업 준비 가이드” 원페이지 웹서비스를 제작해라.

## 0. 가장 중요한 원칙

이 서비스는 법률·세무·플랫폼 정책을 단정적으로 안내하는 서비스가 아니다.  
사용자가 통신판매업/스마트스토어 창업 전에 확인해야 할 항목을 “공식 출처 중심”으로 정리하는 안내형 원페이지다.

다음 원칙을 반드시 지켜라.

- 블로그, 카페, 개인 유튜브, 비공식 요약글을 근거로 사용하지 않는다.
- 정부24, 홈택스/손택스, 공정거래위원회, 소비자24, 국가법령정보센터, 네이버 스마트스토어 고객센터 등 공식 출처 위주로 구성한다.
- 법률·세무·신고 기준은 바뀔 수 있으므로 모든 주요 카드에 “최종 확인일”과 “공식 출처 확인하기” 버튼을 둔다.
- “반드시 된다”, “무조건 면제”, “신고 안 해도 된다” 같은 단정 표현을 피한다.
- 예외가 있을 수 있는 내용은 “공식 기준 확인 필요”, “관할 지자체/세무서 확인 권장”으로 표시한다.
- 세무 상담, 법률 자문, 인허가 대행처럼 보이지 않게 한다.
- 사용자가 실제 신고·등록·제출을 하기 전에는 반드시 공식 사이트에서 최신 내용을 확인하도록 유도한다.

## 1. 서비스 개요

서비스명 후보:
- 스마트스토어 창업 준비 체크
- 통신판매업 준비 가이드
- 온라인 판매 시작 전 체크리스트

최종 표시명은 다음으로 사용한다.

서비스명:
“스마트스토어·통신판매업 창업 준비 체크”

부제:
“사업자등록부터 통신판매업 신고, 소비자보호 기본사항까지 공식 출처 중심으로 확인하세요.”

목표 사용자:
- 스마트스토어를 처음 시작하려는 개인
- 온라인 쇼핑몰/마켓 판매를 준비하는 개인사업자
- 통신판매업 신고가 필요한지 확인하고 싶은 예비 창업자
- 사업자등록, 통신판매업 신고, 구매안전서비스 이용확인증, 환불/반품 규정을 한 번에 보고 싶은 사람

서비스 목표:
- 창업 절차를 5~10분 안에 훑어볼 수 있게 한다.
- 공식 사이트로 이동하기 전 준비물을 확인할 수 있게 한다.
- 사용자에게 법률/세무 판단을 대신해주지 않고, 공식 확인 경로를 안내한다.
- 광고 배너를 넣을 수 있는 정보성 원페이지 구조로 만든다.

## 2. 기술 조건

가능하면 DB 없이 제작한다.

권장 구조:
- HTML + CSS + JavaScript
- 또는 React + Vite 기반 정적 페이지
- 배포 대상: Vercel, Cloudflare Pages, Netlify 등 정적 배포 가능 환경
- 외부 API 호출 없이 정적 데이터 파일로 구성
- 체크리스트 진행 상태는 localStorage에 저장
- 모바일 우선 반응형 UI
- SEO 기본 메타태그 포함
- OG 태그 포함
- 접근성 고려: 버튼, 링크, 대비, 키보드 접근성

프로젝트가 이미 존재하지 않는다면 Vite + React로 생성해도 된다.  
단, 지나치게 복잡한 상태관리 라이브러리나 서버 기능은 사용하지 마라.

원페이지 안에 아래 섹션을 순서대로 구성한다.

### A. Hero Section

- 부제: 사업자등록부터 통신판매업 신고, 소비자보호 기본사항까지 공식 출처 중심으로 확인하세요.
- CTA 버튼 2개:

  - “공식 출처 모아보기”

### B. 5단계 창업 준비 흐름

카드형 UI로 구성한다.

1. 판매 방식 정리
   - 개인 판매인지, 사업자 판매인지
   - 판매 품목이 인허가/제한 품목인지
   - 재고 보유/위탁/해외구매대행 여부

2. 사업자등록 확인
   - 홈택스/손택스 또는 세무서 방문
   - 사업장 주소, 임대차 여부, 업종/업태 확인
   - 필요서류 확인

3. 스마트스토어 가입/사업자 전환
   - 네이버 스마트스토어 판매자 유형 확인
   - 제출 서류 확인
   - 사업자 전환 여부 확인

4. 통신판매업 신고 준비
   - 정부24 또는 관할 시·군·구 신고
   - 구매안전서비스 이용확인증 준비
   - 신고번호 입력/관리

5. 운영 전 소비자보호 기본 점검
   - 교환/반품/환불 안내
   - 청약철회 관련 안내
   - 판매자 정보 표시
   - 허위·과장·기만 광고 금지
   - 주문/배송/CS 대응 기준

각 카드에는:
- 핵심 설명
- 확인할 것 3개
- 공식 출처 버튼
- “확인 완료” 체크박스

### C. 창업 전 체크리스트

체크박스 리스트로 구성하고 localStorage에 저장한다.

체크 항목 예시:

판매 준비:
- 판매할 상품군을 정했다.
- 상품이 인허가/신고/인증/표시 의무 대상인지 확인했다.
- 위탁판매/사입/제조/해외구매대행 여부를 정했다.

사업자등록:
- 사업자등록 필요 여부를 확인했다.
- 사업장 주소 사용 가능 여부를 확인했다.
- 임대차계약서 등 필요서류를 확인했다.
- 업태/종목 후보를 정리했다.

스마트스토어:
- 개인판매자/간이사업자/개인사업자/법인사업자 유형을 확인했다.
- 가입에 필요한 서류를 확인했다.
- 구매안전서비스 이용확인증 발급 위치를 확인했다.

통신판매업:
- 통신판매업 신고 필요 여부를 공식 출처에서 확인했다.
- 정부24 또는 관할 지자체 신고 경로를 확인했다.
- 통신판매업 신고번호 입력/관리 기준을 확인했다.
- 상호, 주소, 전화번호 변경 시 변경신고 필요성을 확인했다.
- 휴업/폐업/영업재개 신고가 별도로 있을 수 있음을 확인했다.

소비자보호:
- 교환/반품/환불 정책을 준비했다.
- 청약철회 기본 기간과 예외 가능성을 확인했다.
- 판매자 정보, 상품정보, 배송비, 반품비 표시 방식을 점검했다.
- 허위·과장 광고 금지 원칙을 확인했다.

### D. 사용자 상황별 빠른 안내

간단한 선택형 UI를 만든다.

질문:
1. 나는 아직 사업자등록을 하지 않았다.
2. 스마트스토어 개인판매자로 먼저 시작하려 한다.
3. 이미 개인사업자등록이 있다.
4. 통신판매업 신고를 해야 하는지 확인하고 싶다.
5. 구매안전서비스 이용확인증이 필요하다.
6. 반품/환불 정책 문구를 준비해야 한다.

각 선택지를 누르면 아래에 안내 카드가 표시되게 한다.

예:
- “사업자등록 전이라면 홈택스/손택스 또는 세무서 안내에서 필요서류를 먼저 확인하세요.”
- “통신판매업 신고 여부는 판매자 유형, 매출/거래 규모, 플랫폼 정책, 법령 기준에 따라 달라질 수 있으므로 정부24·공정위·스마트스토어 공식 안내를 함께 확인하세요.”
- “스마트스토어 구매안전서비스 이용확인증은 스마트스토어 판매자정보 메뉴에서 확인할 수 있다는 공식 안내가 있으므로, 최신 고객센터 안내를 확인하세요.”

### E. 공식 출처 모음

반드시 공식 출처 카드 영역을 만든다.

카드 항목:
1. 홈택스/손택스
   - 용도: 사업자등록 신청, 필요서류 확인
   - 버튼: 공식 사이트에서 확인

2. 정부24
   - 용도: 통신판매업 신고, 변경신고, 휴업·폐업·영업재개 신고
   - 버튼: 공식 민원 확인

3. 공정거래위원회
   - 용도: 통신판매사업자 조회, 전자상거래 소비자보호 안내
   - 버튼: 공식 자료 확인

4. 소비자24
   - 용도: 청약철회, 반품, 피해구제 사례 확인
   - 버튼: 소비자 분쟁 사례 확인

5. 국가법령정보센터
   - 용도: 전자상거래 등에서의 소비자보호에 관한 법률 확인
   - 버튼: 법령 확인

6. 네이버 스마트스토어 고객센터
   - 용도: 판매자 가입서류, 통신판매업 신고번호 입력 기준, 구매안전서비스 이용확인증 확인
   - 버튼: 스마트스토어 공식 안내 확인

출처 데이터는 src/data/sources.js 또는 src/data/sources.ts 파일로 분리한다.

각 데이터 구조 예시:

{
  id: "gov24-mail-order",
  title: "통신판매업 신고",
  organization: "정부24",
  category: "신고/민원",
  description: "통신판매업 신고 민원 안내 및 신청 경로",
  url: "공식 URL",
  lastChecked: "2026-04-30",
  caution: "관할 지자체 및 신청자 상황에 따라 필요서류가 달라질 수 있습니다."
}

### G. FAQ

질문 예시:

Q1. 사업자등록을 먼저 해야 하나요?
A. 일반적으로 사업자 판매를 준비한다면 사업자등록 여부를 먼저 확인해야 합니다. 다만 개인 판매자, 플랫폼 정책, 판매 규모 등에 따라 절차가 달라질 수 있으므로 홈택스와 스마트스토어 공식 안내를 확인하세요.

Q2. 통신판매업 신고는 어디서 하나요?
A. 정부24 또는 관할 시·군·구청 안내를 통해 확인할 수 있습니다. 신고 대상과 필요서류는 상황에 따라 달라질 수 있습니다.

Q3. 구매안전서비스 이용확인증은 무엇인가요?
A. 통신판매업 신고 과정에서 요구될 수 있는 서류입니다. 스마트스토어의 경우 판매자센터 내 공식 안내에서 발급 경로를 확인하도록 안내하세요.

Q4. 반품불가라고 써도 되나요?
A. 전자상거래에서는 청약철회와 소비자보호 규정이 적용될 수 있으므로 단순한 “반품불가” 고지는 위험할 수 있습니다. 소비자24, 공정거래위원회, 국가법령정보센터 자료를 확인하도록 안내하세요.

Q5. 이 서비스만 보고 신고해도 되나요?
A. 아니요. 이 서비스는 준비 체크를 돕는 정보성 페이지입니다. 실제 신고·등록·서류 제출 전에는 반드시 공식 사이트에서 최신 내용을 확인해야 합니다.

## 7. 공식 출처 URL 입력 방식

소스 코드에 공식 출처 URL을 넣되, 반드시 주석으로 “정책 변경 가능, 정기 확인 필요”를 표시한다.

공식 출처 후보:
- 정부24: 통신판매업 신고 / 변경신고 / 휴업·폐업·영업재개 신고
- 홈택스/손택스: 개인사업자등록 신청 / 필요서류
- 공정거래위원회: 통신판매사업자 조회 / 전자상거래 소비자보호
- 소비자24: 청약철회, 반품, 소비자 피해/분쟁 사례
- 국가법령정보센터: 전자상거래 등에서의 소비자보호에 관한 법률
- 네이버 스마트스토어 고객센터: 판매자 가입서류 / 통신판매업 신고번호 입력 기준 / 구매안전서비스 이용확인증

주의:
- URL과 기준 문구는 코드 작성 시 최신 공식 페이지를 기준으로 확인한다.
- 비공식 블로그 URL은 넣지 않는다.
