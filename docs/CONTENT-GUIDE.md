# 콘텐츠 관리 가이드

> 스마트스토어·통신판매업 창업 준비 체크 서비스의 콘텐츠를 안전하게 관리하기 위한 가이드입니다.

---

## 1. 핵심 원칙

이 서비스는 **법률·세무·플랫폼 정책을 단정적으로 안내하는 서비스가 아닙니다.**  
사용자가 통신판매업/스마트스토어 창업 전에 확인해야 할 항목을 "공식 출처 중심"으로 정리하는 안내형 원페이지입니다.

### 반드시 지켜야 할 원칙

1. **출처 기준**: 블로그, 카페, 개인 유튜브, 비공식 요약글을 근거로 사용하지 않습니다.
2. **공식 출처 우선**: 정부24, 홈택스/손택스, 공정거래위원회, 소비자24, 국가법령정보센터, 네이버 스마트스토어 고객센터 등 공식 출처 위주로 구성합니다.
3. **확인 유도**: 법률·세무·신고 기준은 바뀔 수 있으므로 모든 주요 카드에 "최종 확인일"과 "공식 출처 확인하기" 버튼을 둡니다.
4. **단정 금지**: "반드시 된다", "무조건 면제", "신고 안 해도 된다" 같은 단정 표현을 피합니다.
5. **예외 표시**: 예외가 있을 수 있는 내용은 "공식 기준 확인 필요", "관할 지자체/세무서 확인 권장"으로 표시합니다.
6. **오해 방지**: 세무 상담, 법률 자문, 인허가 대행처럼 보이지 않게 합니다.
7. **최신 확인 유도**: 사용자가 실제 신고·등록·제출을 하기 전에는 반드시 공식 사이트에서 최신 내용을 확인하도록 유도합니다.

## 3. 공식 출처 URL 관리 (`sources.js`)

### URL 점검 체크리스트

각 출처의 URL을 점검할 때 다음을 확인하세요:

- [ ] URL이 실제로 해당 공식 기관의 URL인가?
- [ ] 페이지에 접속했을 때 안내 내용이 최신인가?
- [ ] URL이 변경되거나 리디렉션이 발생했는가?
- [ ] 기관명·조직 명칭이 변경되지 않았는가?

### `lastChecked` 필드 업데이트 규칙

URL 또는 내용을 확인한 날짜를 `YYYY-MM-DD` 형식으로 업데이트합니다:

```js
lastChecked: "2026-04-30",  // 실제 확인한 날짜로 업데이트
```

카드형 콘텐츠 메타는 다음 규칙으로 관리합니다:

- `src/data/sources.js`: 각 출처 객체의 `lastChecked`를 직접 갱신합니다.
- `src/data/steps.js`, `src/data/quickGuides.js`: 파일 상단의 `CONTENT_LAST_CHECKED` 상수를 갱신하고 각 카드의 `lastChecked`에 연결합니다.

```js
const CONTENT_LAST_CHECKED = "2026-04-30";

{
  id: "step-1",
  // ...
  lastChecked: CONTENT_LAST_CHECKED,
}
```

## 5. 콘텐츠 추가/수정 절차

### 출처 카드 추가 시

1. `src/data/sources.js` 열기
2. 아래 템플릿으로 항목 추가:
   ```js
   {
     id: "unique-id",           // 고유 ID (영문 소문자, 하이픈)
     title: "출처 제목",
     organization: "기관명",
     category: "카테고리",      // 사업자등록 | 통신판매업 | 소비자보호 | 스마트스토어 | 법령
     description: "출처 설명",
     url: "https://공식URL",    // 정책 변경 가능, 정기 확인 필요
     lastChecked: "YYYY-MM-DD",
     caution: "주의사항",       // 선택사항
   }
   ```
3. 비공식 출처인지 반드시 확인 후 추가
4. `lastChecked`를 오늘 날짜로 설정

### FAQ 추가 시

1. `src/data/faq.js` 열기
2. 아래 형식으로 추가:
   ```js
   {
     id: "faq-N",
     question: "질문 내용?",
     answer: "답변 내용 (공식 출처 확인 유도 포함)",
   }
   ```
3. 단정적인 법률/세무 판단을 포함하지 않도록 주의

### 단계 카드/빠른 안내 수정 시

1. `src/data/steps.js` 또는 `src/data/quickGuides.js` 열기
2. 카드 내용, 공식 출처 링크, 주의 문구를 수정
3. 변경이 반영되었다면 파일 상단 `CONTENT_LAST_CHECKED` 날짜를 오늘 날짜로 갱신
4. `src/App.jsx`에서 카드 하단의 `최종 확인일` 노출이 유지되는지 확인
5. `npm run build`로 빌드 확인

### 체크리스트 항목 추가 시

1. `src/data/checklist.js` 열기
2. 기존 카테고리에 항목 추가 또는 새 카테고리 생성
3. 항목 ID는 `카테고리접두사-번호` 형식 유지 (예: `sp-4`, `br-5`)

## 6. 정기 점검 체크리스트

### 분기별 점검 (최소)

- [ ] 모든 공식 출처 URL 접속 가능 여부 확인
- [ ] URL 변경 또는 리디렉션 여부 확인
- [ ] 전자상거래법 및 관련 고시 개정 여부 확인
- [ ] 공정거래위원회 전자상거래 관련 공지 확인
- [ ] 스마트스토어 정책 변경 공지 확인
- [ ] `lastChecked` 날짜 업데이트
- [ ] `steps.js`, `quickGuides.js`의 `CONTENT_LAST_CHECKED` 날짜 업데이트
- [ ] 부정확한 문구나 만료된 안내 제거

### 즉시 점검 필요 시

- 법령 개정 발표 시
- 공정거래위원회 고시 변경 시
- 스마트스토어 주요 정책 변경 공지 시
- 통신판매업 신고 절차 변경 시
- 사용자 피드백으로 오류 발견 시

## 7. 광고 영역 관리

현재 광고 영역은 **Placeholder** 상태입니다. 실제 광고 연동 시:

- Hero 섹션 아래: `<AdBanner position="top" />`
- FAQ 섹션 위: `<AdBanner position="mid" />`

`src/App.jsx`의 `AdBanner` 컴포넌트 내부에 광고 스크립트를 삽입합니다.

> ⚠️ 광고 영역은 정보성 콘텐츠를 방해하지 않는 수준으로 유지하세요.
